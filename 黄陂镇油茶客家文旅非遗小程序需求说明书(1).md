# 一、引言

## （一）编写目的

本文档用于明确黄陂镇油茶+客家文旅非遗一体化微信小程序一期建设的业务范围、功能需求、接口需求、数据要求、非功能要求、实施计划与验收标准，作为项目开发、测试、验收、上线和后续迭代的统一依据。

本文档特别强调4人团队与8周周期下的可落地性，避免将一期建设扩大为大型综合数字乡村平台。所有功能均按照“优先上线、稳定可用、后台可维护、线下可承接”的原则进行拆分。

## （三）预期读者

- 项目甲方、乡镇主管部门及平台运营负责人：用于确认建设范围、交付边界和验收口径。

## （四）项目范围

一期建设范围限定为微信小程序用户端、轻量化后台管理端、基础内容数据与预约业务闭环。系统定位为“乡村特色资源展示与线下预约服务小程序”，重点支撑资源信息展示、联系方式查询、线下体验预约和后台内容维护，不包含线上经营闭环、直播或复杂社区能力。

| 范围内功能 | 范围外功能 |
| --- | --- |
| 首页展示、资源导航、活动公告 | 复杂会员体系、直播互动、算法推荐、短视频运营 |
| 油茶产业、客家文旅、非遗文化、本地特产展示 | 本地特产展示并指引线下参考 |
| 非遗体验、研学活动、基地参观预约申请 | 复杂社区发帖、评论点赞回复、私信 |
| 后台轮播图、内容、预约、公告和基础统计管理 | 会员积分、精准推荐、大数据画像、复杂报表 |
| 已有文创IP素材的页面适配与轻量动效 | 原创IP设计、复杂剧情互动 |

# 二、综合描述

## （一）项目背景

黄陂镇拥有油茶产业、客家文化、非遗技艺、乡村旅游、本地特产等资源，但资源信息分散、线上展示入口不统一、游客获取信息不便利、线下体验预约缺少规范流程。建设微信小程序有助于将分散资源形成统一入口，提升对外宣传效率和线下服务组织能力。

本项目一期不追求“大而全”，而是通过可上线的MVP版本完成“看得见资源、找得到入口、约得上线下活动、管得住后台内容”的基础闭环。

## （二）产品定位

| 维度 | 定位说明 |
| --- | --- |
| 产品形态 | 微信小程序用户端 + PC/网页后台管理端 |
| 服务对象 | 游客、本地居民、研学团队、非遗体验参与者、平台运营人员 |
| 核心价值 | 统一展示黄陂镇特色资源，降低游客获取信息和预约线下体验的成本 |
| 交付策略 | 一期MVP先上线，后续根据运营数据扩展线下活动组织、互动服务和数据分析能力 |
| 团队约束 | 4人团队开发，不设置过多复杂交互和高并发后台体系<br>可使用codex、figma、cursor、claude code辅助开发 |

## （六）假设和依赖

- 项目甲方需在第2周前提供基础内容资料，包括油茶基地、非遗项目、文旅资源、特产资料、图片素材和联系方式。

- 文创IP素材为已有成品素材，本项目只负责界面适配、裁切、排版和轻量动效，不负责原创设计。

- 线下预约最终接待能力由运营方确认，小程序只负责预约申请、审核和状态展示。

- 微信小程序主体、备案、服务类目、服务器域名等上线资料需由甲方或运营方配合提供。

# 五、功能需求

## （一）功能优先级定义

| 优先级 | 定义 | 验收处理 |
| --- | --- | --- |

## （一）视觉与IP素材使用要求

- 使用已有成品IP素材，不开展原创IP形象设计。

- IP元素主要用于首页装饰、空状态提示、分区引导、按钮点缀和封面视觉，不影响内容阅读。

- 动效以轻量为原则，包括悬浮、淡入、点击反馈等，不开发复杂互动剧情。

- 图片资源应压缩后上传，避免造成页面卡顿。

## （三）变更控制要求

# 十三、风险控制与二期规划

## （一）风险控制

| 风险 | 表现 | 影响 | 控制措施 |
| --- | --- | --- | --- |
| 功能范围膨胀 | 新增社区、算法等复杂需求 | 延期或质量下降 | 一期冻结MVP范围，新增需求进入二期池 |
| 资料准备不足 | 图片、文字、地址、联系方式不完整 | 页面空缺、上线延期 | 第2周前完成资料清单确认，缺失内容用占位数据标记 |
| IP素材适配耗时 | 素材尺寸不统一、风格不一致 | UI返工 | 只做现有素材适配，不做原创IP设计 |
| 后台复杂度过高 | 多角色、多流程、多审核链 | 开发成本增加 | 一期只保留管理员、内容维护人员两级为主 |
| 微信审核延期 | 服务类目、隐私协议、域名配置不完整 | 上线时间不确定 | 第7周提前准备小程序审核资料 |
| 线下预约承接不清 | 审核通过后无人联系用户 | 用户体验下降 | 明确运营方责任，后台保留联系方式和备注字段 |

## （二）二期规划

| 方向 | 可扩展内容 | 触发条件 |
| --- | --- | --- |
| 线下服务承接 | 线下咨询、到访路线、电话联系、现场体验引导 | 特产展示有明确线下咨询需求并具备运营承接人员 |
| 活动能力 | 研学课程报名、活动签到、名额管理 | 线下活动频次提升，手工处理成本变高 |
| 互动能力 | 评论、点赞、用户投稿、社区动态 | 已有稳定内容运营和审核机制 |
| 会员能力 | 会员档案、复访提醒、权益说明、活动通知 | 有持续复访和消费场景 |
| 数据能力 | 热度排行、用户画像、预约转化分析 | 一期积累足够访问和预约数据 |
| IP互动 | IP角色引导、动画剧情、文创定制 | 已有完整IP运营策略和素材预算 |

二期功能不得写入一期验收范围。二期建设应基于一期真实运营数据和团队承接能力进行评估。
