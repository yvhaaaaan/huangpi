# 1. 项目说明

黄陂镇特色产业客家文旅非遗小程序是一款特色产业展示和文旅导览类微信小程序。它不是电商平台，主要目标是让用户快速了解黄陂镇有什么特色产品、在哪里、怎么玩、有哪些活动可以参加。

前端页面需要做到结构清楚、入口明确、图片区域有占位、内容为空时不乱版。油茶和丝苗米均为一级重点产品品类。普通用户主要浏览和报名；商家只使用“我的”和“新增”维护自己的特色产品；政府管理员使用工作台和审核页处理商家内容。

# 5. 全局前端规则

## 5.1 登录规则

所有用户打开小程序后必须先登录，未登录用户只能访问统一登录页。

普通用户使用微信登录；商家和政府管理员使用账号密码登录。账号密码登录页不允许手动选择角色。

登录成功后，前端根据后端返回的 `user.role` 直接进入对应角色首页。退出登录后清除本地会话并返回登录页。

本地 token 存在时，真实接口模式调用 `/api/auth/session` 校验；后端返回 `401` 或业务码 `40100` 时自动清除会话并重新登录。

## 5.2 图片占位规则

首页轮播图、卡片封面、详情页轮播图、活动封面、特产图片都需要预留固定图片区域。

后台未上传图片时，前端显示默认占位图。

图片未审核通过时，前台不展示真实图片。

图片加载失败时，显示默认占位图，避免页面空白或布局错乱。

## 5.3 内容状态规则

| 状态 | 含义 | 前端展示规则 |
| --- | --- | --- |
| 草稿 | 内容还未提交 | 只在管理页面显示，普通前台不显示。 |
| 待审核 | 商家提交后等待审核 | 只在管理页面显示，普通前台不显示。 |
| 已发布 | 审核通过 | 前台可以展示。 |
| 已驳回 | 审核未通过 | 商家管理页面显示驳回原因，前台不显示。 |
| 已下架 | 内容不再展示 | 前台不显示，收藏列表可提示“内容已下架”。 |

## 5.4 收藏规则

普通用户可以收藏油茶文化、特色产品、景点、非遗项目、路线和活动。

未收藏时显示普通收藏图标，已收藏时图标高亮。

接口返回未登录时，统一请求层回到登录页。

内容下架后，收藏列表中可隐藏该内容，或显示“内容已下架”。

## 5.5 查看更多规则

首页各推荐栏目默认展示最多 3 条。

特色产品展示页首页区域默认展示前 3 条，油茶和丝苗米按同级重点品类处理。

当内容超过 3 条时，显示“查看更多”按钮。

点击“查看更多”进入对应列表页，展示全部已发布内容。

## 5.6 后端接口预留与联调规则

前端必须通过统一接口层访问后端，不允许在页面文件中散写 `wx.request` 或后端域名。

| 文件 | 责任 |
| --- | --- |
| `miniprogram/config/api.ts` | 配置 `baseUrl`、`useMock` 和请求超时时间。 |
| `miniprogram/utils/request.ts` | 统一封装 `wx.request`、返回结构、错误对象、token 注入和 `401` 处理。 |
| `miniprogram/api/auth.ts` | 微信登录、账号密码登录、会话校验、退出登录。 |
| `miniprogram/api/products.ts` | 产品品类、产品列表和产品详情。 |
| `miniprogram/api/user.ts` | 首页、搜索、地图、路线、活动、收藏和报名。 |
| `miniprogram/api/merchant.ts` | 商家工作台、资料、产品、消息和待办。 |
| `miniprogram/api/admin.ts` | 政府工作台、审核、商家、品类和操作日志。 |
| `miniprogram/api/files.ts` | 图片上传、文件查询和逻辑删除。 |

联调规则：

- 当前开发阶段保持 `useMock: true`，页面继续使用本地模拟数据和测试账号。
- 后端可用后，将 `baseUrl` 改为已配置到微信小程序合法域名中的 HTTPS 地址，并把 `useMock` 改为 `false`。
- 请求层自动从 `auth.session.v2` 读取 token，并添加 `Authorization: Bearer <token>`。
- 普通接口统一解析 `{ code, message, data, traceId }`，分页接口统一解析 `{ list, page, pageSize, total }`。
- 更新接口使用 `PUT`，不使用微信小程序请求类型未声明的 `PATCH`。
- 图片上传使用 `wx.uploadFile`，字段名为 `file`，附带 `businessType`。
- 后端返回 HTTP `401` 或业务码 `40100` 时，前端清除登录信息并 `reLaunch` 到登录页。
- 页面需要处理加载中、空数据、网络失败和业务错误，不直接展示后端原始异常堆栈。

# 7. 商家管理前端需求

商家登录后直接进入商家端，不经过普通用户首页。商家端一期只保留“我的”和“新增”两个页面，不提供活动发布、活动报名查看或政府审核功能。
